**기준일**: 2026-04-15
**기반 DB**: `content-input/keywords/*.json` 실측 키워드 (2,333개+)
**대상 어필리에이트**: 법인설립지원센터, 키퍼메이트(CCTV), 노무(HR 계산기 교차용)

## 절대 원칙 (변경 금지)

1. **목표는 오직 유입과 노출이다.** 발행 편수·CTA 위치·어필리에이트 배분은 2차.
2. 모든 의사결정 판단 기준: "이 글이 검색 노출되고 유입을 만드는가?"
3. "검색량이 크면 좋다"는 함정이다. `comp=high` + 검색량 10만 = 우리가 노출될 수 없는 영역.
4. 실제 달성 가능한 노출 = `comp=low/medium` 키워드 + 의문문 롱테일로 진입.
5. 허브 키워드(검색량 큰 것)는 내부링크 허브 역할만 맡고, 실제 트래픽은 롱테일로 확보.
6. 모든 주제는 "6개월 내 100 유입 이상 기대 가능한가?" 필터 통과 필수. 통과 못 하면 발행하지 않는다.

## 키워드 분류 매트릭스

NAVER 키워드도구 `comp`는 **광고 경쟁**이지 **SEO 경쟁**이 아니다. 해석 기준:

| comp | 의미 | 우리 전략 |
|------|------|-----------|
| `low` | 광고 입찰 희박 = 정보성 검색 주류 | 최우선 타겟. 본문 + FAQ로 직접 진입 가능 |
| `medium` | 광고 중간 경쟁 = 상업+정보 혼합 | 2순위 타겟. 비교·가격·후기 앵글로 진입 |
| `high` | 광고 포화 = 상업 의도 높음 | 단독 허브 주제로 쓰되, 실제 노출은 의문문 롱테일 H2에서 확보 |

### 검색량 구간 전략

| total 구간 | 해석 | 역할 |
|-----------|------|------|
| `>= 10,000` | 대형 허브 | 허브 1편, 모든 내부링크의 종착지 |
| `1,000 ~ 10,000` | 핵심 주제 | 각 클러스터의 1순위 제목 |
| `100 ~ 1,000` | 롱테일 본진 | 클러스터당 3~5편 제작 대상. 실제 유입 주력 |
| `40 ~ 100` | 롱테일 끝단 | H2·FAQ 질문 텍스트로 흡수 |
| `< 40` | 무시 | 단, 의문문 자연어 검색 대응 목적이면 FAQ에 추가 가능 |

## 허브 - 롱테일 클러스터 구조

### 기본 단위: 허브 1편 + 롱테일 3~5편

```
[허브 포스트]   키워드 vol 1,000+ (comp 무관)
    │
    ├─ [롱테일 1]  vol 100~1,000 + comp low/medium
    ├─ [롱테일 2]  vol 100~1,000 + comp low/medium
    ├─ [롱테일 3]  vol 100~1,000 + comp low/medium
    └─ [롱테일 4]  vol 40~200   + 의문문 자연어

허브 → 롱테일: "관련 글" 섹션에서 전부 링크
롱테일 → 허브: 본문 중간 1회 + 정리 섹션 1회 링크
롱테일 ↔ 롱테일: 같은 클러스터끼리 1~2개씩 상호 링크
```

이 구조는 Google topical authority, 네이버 C-Rank 문맥성 두 축을 동시 충족한다.

## 신규 콘텐츠 기획 체크리스트

주제를 `topic-queue.json`에 넣기 전 반드시 아래 7개 항목을 통과해야 한다.

- [ ] **1. 키워드 DB 매칭 확인**: `content-input/keywords/*.json`에 동일 또는 유사 키워드가 존재하는가? (`scripts/discover-topics.mjs` 또는 `scripts/enqueue-topics.mjs` 경유)
- [ ] **2. bestMatch.total >= 100**: 매칭된 키워드의 월 검색량이 최소 100 이상인가? (매우 롱테일 자연어는 예외, 단 FAQ 흡수용일 경우에만)
- [ ] **3. bestMatch.comp 전략 대응**: comp=high라면 의문문 H2 롱테일 진입 전략이 명확한가? low/medium이라면 허브 링크 구조가 준비됐는가?
- [ ] **4. 클러스터 내 위치 확정**: 이 글이 허브인가 롱테일인가? 상위 허브는 무엇인가? 동일 클러스터 내 다른 롱테일과 어떻게 링크되는가?
- [ ] **5. 의문문 H2 쿼리 4개 이상 확보**: 의문문 H2 비율 40%+ 달성 가능한가? 위 쿼리 뱅크에서 선택 또는 유사 쿼리 신규 작성.
- [ ] **6. 6개월 100유입 필터**: 이 주제의 검색량·경쟁도·내부링크 구조로 볼 때, 발행 6개월 후 월 100유입이 기대되는가? 기대 못 하면 발행하지 않는다.
- [ ] **7. opportunityScore 확인**: `discover-topics.mjs` 산출 `opportunityScore >= 20`인가? 미만이면 `--force-unscored` 없이는 등록 금지.
