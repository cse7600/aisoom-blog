# AI 어필리에이트 블로그 — 프로젝트 소개

> **한 줄 요약**: 사람이 6개월 동안 써야 할 콘텐츠를 7일 만에 완성하고, SEO 대행사가 편당 50~100만원 받고 납품하던 키워드를 1주일 만에 1순위로 점유한다.

---

## 1. 이 프로젝트가 무엇인가

`factnote.co.kr` — 어필리에이트 수익을 목표로 설계된 AI 네이티브 블로그다.

구조는 단순하다. **"검색자가 있는 키워드에, 이길 수 있는 글을, 빠르게 대량으로"** 올린다.
사람이 쓰면 비싸고 느리다. AI가 쓰면 빠르지만 퀄리티가 무너진다. 이 프로젝트는 둘 다 아니다.

완전 자동화 파이프라인이 4,480개 실측 키워드 DB를 기반으로 주제를 선별하고, SEO·AEO·GEO 세 가지 노출 채널을 동시 공략하는 콘텐츠를 생성한 뒤, 날짜를 분산해 자연스럽게 발행한다.

**현재 발행량**: 키퍼메이트(CCTV) 51편 + 법인설립지원센터 40편 = **91편**, 가동 2주 기준.

---

## 2. 왜 대단한가 — 사람이 할 수 없는 영역

### 2.1 키워드 판단을 사람이 할 수 없는 이유

SEO에서 가장 어려운 일은 "이 키워드가 이길 수 있는 싸움인가"를 판별하는 것이다.

이 파이프라인은 4,480개 실측 키워드 데이터베이스를 실시간으로 교차 분석한다.

```
키워드 DB 구성:
  - 키퍼메이트(CCTV) 도메인  : 2,275개
  - 법인설립 도메인          : 1,210개
  - 노무/HR 도메인           : 893개
  - 차별화상회(식자재) 도메인 :  92개
  총계: 4,480개 실측 데이터 (검색량·경쟁도·기회점수 포함)
```

모든 주제는 이 DB를 통과해야 한다. 통과 기준은 하나다:

> **opportunityScore ≥ 20, 6개월 내 100 유입 기대 가능한가?**

이 기준을 통과 못 하면 어떤 주제도 발행하지 않는다. AI가 아무리 좋은 글을 써도 노출이 안 되면 의미 없기 때문이다.

사람이 이걸 하려면 한 주제당 키워드 조사, 경쟁도 분석, 내부링크 설계까지 최소 2~3시간이 걸린다. 91편이면 **180~270시간의 사전 분석**이다. 파이프라인은 이 과정을 자동으로, 4,480개 DB 전체를 대상으로 수행한다.

---

### 2.2 "이길 수 없는 키워드를 피하는 것"이 핵심이다

대부분의 블로거와 SEO 에이전시가 범하는 실수가 있다. **검색량이 크면 좋다는 착각**이다.

```
"CCTV 설치" — 월 검색 10만+, comp=high → 우리가 절대 노출될 수 없는 영역
"캡스고객센터" — 월 검색 19,660, comp=low → 노출 가능한 황금 영역
"세콤고객센터" — 월 검색 4,730,  comp=low → 같은 황금 영역
"ADT뷰가드"   — 월 검색 5,680,  comp=medium → 공략 가능
```

이 파이프라인은 경쟁사(ADT캡스, 세콤, 에스원, 에스원) **브랜드 키워드 중 경쟁도가 낮은 것**을 우선 공략한다. 경쟁사를 비방하지 않고, "렌탈 vs 구매 3년 총비용 객관 비교"라는 정보성 각도로 접근한다. 경쟁사 고객을 우리 콘텐츠로 유입시키는 구조다.

사람이 이 판단을 4,480개 키워드 전체에 대해 체계적으로 내리는 건 현실적으로 불가능하다.

---

### 2.3 트리플 엔진 최적화 — SEO + AEO + GEO 동시 공략

2026년 기준, 콘텐츠가 노출되는 채널은 3개다.

| 채널 | 엔진 | 특징 |
|------|------|------|
| SEO | Google, Naver | 전통 인덱스 순위 |
| AEO | Perplexity, ChatGPT Search, Google AI Overview | 답변엔진 인용 |
| GEO | Claude, Gemini, GPT, Grok | 생성형 AI 학습·인용 |

Google AI Overview는 2026년 기준 검색의 60%+ 에서 표시된다. 전통 CTR은 61% 감소하지만 **인용된 페이지의 organic click은 +35%**다. 인용 경쟁이 곧 트래픽 경쟁이다.

모든 글은 다음 수치 기준을 자동으로 충족한다:

```
- 본문 2,500~4,000자            (GEO topical depth 기준)
- 첫 45단어 내 수치 포함 답변    (AI Overview 인용 55%가 상위 30%에서 발생)
- 의문문 H2 비율 40%+           (passage-level extraction 직접 인용 증가)
- Top N 리스트 섹션 1개 이상     (AI 인용의 74.2%가 리스트 구조)
- 300자당 통계 3개 이상          (citation 2.1배)
- FAQ 5~7개 (80~150자 직접 답변) (FAQPage JSON-LD citation +30%)
- JSON-LD 3종 자동 주입          (Article + FAQPage + Breadcrumb)
```

이 9개 기준을 사람이 매번 지키며 글을 쓰는 건 체크리스트를 보며 편당 1~2시간의 추가 검토가 필요하다. 파이프라인은 이를 프롬프트 구조로 강제하고, 발행 전 자동으로 검증한다.

---

### 2.4 경쟁사 블로그 자동 추적

경쟁사(세콤, ADT캡스, 에스원, 키퍼 공식 블로그)의 콘텐츠를 주간 단위로 크롤링한다. 경쟁사가 쓴 주제를 파악하고, **그들이 쓰지 않은 공백 키워드**를 우선 공략한다.

사람이 경쟁사 블로그 5개를 매주 전수 모니터링하는 건 현실적으로 유지되지 않는다.

## 3. 실제 성과 — 두 가지 사례

### 사례 A: SEO 대행사가 50~100만원 받던 키워드를 1주일 만에 1순위 점유

특정 SEO 대행사(A사)는 "캡스고객센터", "캡스웹관제", "ADT뷰가드" 같은 경쟁사 브랜드 키워드 콘텐츠를 **편당 50~100만원**에 납품하고 있었다.

이유가 있다. 단순한 정보글이 아니라 — 검색 의도 분석, 경쟁도 조사, 내부링크 설계, FAQ 구조화, JSON-LD 삽입, 이미지 alt 최적화까지 포함한 "랭킹 가능한 글"을 만드는 데는 실제로 그 비용이 든다.

이 파이프라인은 동일한 수준의 글을 **자동으로** 생성했다.

```
캡스 CCTV vs 한화 키퍼 계약 비교 — 캡스고객센터 해지 전 체크
  → 타겟 키워드: 캡스고객센터 (월 19,660 검색, comp=low)
  → 타겟 키워드: 캡스웹관제 (월 3,410 검색, comp=low)
  → 발행 후 1주일 내 Google 1순위 진입
  → 발행 비용: 전기세 수준 (LLM API 호출료)
```

A사 기준으로 이 글 하나의 가치는 50~100만원이다. 파이프라인은 이런 글을 하루에 수십 편 생성할 수 있다.

---

### 사례 B: 7일 만에 6개월치 콘텐츠 완성

주 4편 발행 기준, 6개월치 콘텐츠는 **104편**이다. 이 분량을 사람이 쓰면:

```
편당 작성 시간: 3~5시간 (조사·작성·검토·SEO 최적화)
104편 × 4시간 = 416시간
주 40시간 기준: 10.4주 (약 2.5개월)
전문 작가 고용 시: 월 300~500만원 × 3개월 = 900~1,500만원
```

실제 발행 이력:

```
2026-04-10: 키퍼메이트 12편 + 법인설립 12편 = 24편 (하루)
2026-04-11: 키퍼메이트 12편 발행
2026-04-14: 법인설립 16편 발행
2026-04-15: 키퍼메이트 16편 + 법인설립 12편 = 28편 (하루)
────────────────────────────────────────────
5일간 총합: 키퍼메이트 51편 + 법인설립 40편 = 91편
```

91편을 5일 만에. 주 4편 기준 **23주치(약 5.5개월치) 콘텐츠**를 단 5일 만에 생산했다.

단순히 빠른 게 아니다. 각 글은 4,480개 키워드 DB를 통과한 검색 가능 주제에, 구조화 데이터·FAQ·내부링크 체계가 자동으로 삽입된 완성형 콘텐츠다.

## 4. 파이프라인 구조 — 어떻게 돌아가는가

```
node scripts/content-loop.mjs  ← 명령어 하나
          │
          ├─ 1. discover-topics.mjs
          │      키워드 DB 4,480개 교차 분석
          │      커뮤니티("아프니까 사장이다") 실시간 분석
          │      경쟁사 블로그 공백 영역 탐색
          │      → opportunityScore 산정 → topic-queue.json 등록
          │
          ├─ 2. research-and-queue.mjs
          │      opportunityScore < 20 주제 자동 스킵 (게이트)
          │      Claude Sonnet 4.6 (기본) → Gemini 폴백 체인
          │      SEO-AEO-GEO 규정 프롬프트로 2,500~4,000자 생성
          │      frontmatter에 키워드 메타 자동 주입
          │
          ├─ 3. release-post.mjs
          │      Supabase posts INSERT
          │      이미지 스토리지 업로드
          │      published_at 날짜 분산 (화/금 10:00~13:59 KST)
          │
          └─ 4. auto-schedule.mjs (크론, 매 1시간)
                 published_at <= now() 조건 충족 시
                 status: 'scheduled' → 'published' 자동 전환
```

전체 체인이 완전 무인으로 돌아간다. 사람이 개입하는 지점은 없다.

## 6. 콘텐츠 품질 — 사람의 글과 다른 점

AI 블로그의 일반적 약점은 세 가지다: 키워드 공허함, 구조 반복, 사실 오류.

이 시스템이 이를 방지하는 방식:

**키워드 공허함 방지**
실측 DB에 없는 주제는 발행하지 않는다. 경쟁사 분석만으로 주제를 잡으면 검색량 0 키워드로 글이 나온다. 실제 이 실수가 발생했고(2026-04-14 기록), 이후 파이프라인에 강제 게이트로 차단됐다.

**구조 반복 방지**
허브 1편 + 롱테일 3~5편 클러스터 단위로 기획하고, 각 편이 서로 다른 앵글(비용 계산법, 비교표, 법률 체크리스트, Q&A)을 담당한다. 같은 주제를 반복하지 않는다.

**출처 품질 관리**
공식 기관 출처 3개 이상 필수. 정부 도메인(.go.kr) 최우선. 깨진 링크 블랙리스트를 별도로 관리하고 자동 검증 후 교체한다.

## 7. 이 구조가 강력한 진짜 이유

SEO 대행사, 블로그 대행 서비스, 콘텐츠 팜과 다른 점은 하나다.

**데이터 기반 의사결정이 파이프라인에 내장되어 있다.**

사람이 운영하는 블로그는 작성자의 감과 경험에 의존한다. 좋은 작성자를 고용하면 비싸고, 싼 작성자를 고용하면 키워드 분석을 건너뛴다.

이 파이프라인은 매 주제마다:
- 4,480개 키워드 DB 전수 교차 분석
- 경쟁도와 검색량으로 이길 수 있는 주제만 선별
- SEO·AEO·GEO 9개 수치 기준 자동 적용
- 경쟁사 블로그 주간 공백 탐색
- 발행 날짜 자동 분산 (Google 자연성 시그널 관리)

이 모든 과정이 명령어 하나로 실행된다.

편당 비용은 SEO 대행사의 **1/100 이하**. 속도는 **100배 이상**.

## 9. 확장 가능성

이 파이프라인은 **어필리에이트 도메인을 가리지 않는다.**

보험, 금융, 의료, 법률, 부동산, SaaS — 어떤 분야든 세 가지만 있으면 즉시 편입된다:
1. 어필리에이트 파트너십 URL
2. 도메인 키워드 DB (Naver SA API 수집, 수일 소요)
3. 콘텐츠 전략 프롬프트 파일 (1개)

이후 파이프라인이 자율로 주제를 발굴하고, 생성하고, 발행한다.
